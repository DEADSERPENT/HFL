1.1  Original Problem Statement (from D3, verbatim)

  "The proposed framework implements a Hierarchical Federated Learning
  (HFL) architecture operating across three layers: IoT/End Devices,
  Edge Servers, and a Cloud Server. Unlike traditional flat federated
  learning, the system adopts a two-tier aggregation strategy that
  balances local privacy preservation with global model accuracy."

1.3  Original Architecture Planned (from D3, Figure 1)

  Layer 1: IoT/End Devices
    → Multimodal data collection (image, audio, vibration, env. sensors)
    → On-device feature extraction (lightweight)
    → Differential Privacy noise injection (ε ≤ 1)
    → Local model training (only updates leave device)
    → Encrypted local updates {W_i, W_j}

  Layer 2: Edge Server Layer
    → Dynamic device clustering
    → Edge aggregation (FedAvg at edge)
    → Update filtering + compression
    → Local model redistribution to devices

  Layer 3: Cloud Server
    → Final global aggregation (FedAvg)
    → Global model W_global distributed back to edges

  Data flow: Uplink: encrypted gradients | Downlink: global model

1.4  Original Methodology (from D3, Section 0.4)

  Stage 1: Privacy-Preserving Local Training
    "Calibrated noise (Gaussian or Laplace) added to local gradients
    to achieve gradient indistinguishability. Raw data never leaves
    the device."

  Stage 2: Hierarchical Aggregation
    Edge: "Dynamically clustering devices, partial aggregation — targets
          up to 50% reduction in uplink traffic."
    Cloud: "Final aggregation on compressed, privacy-safe updates."

  Stage 3: QoS Validation
    "EdgeCloudSim or NS-3 for simulation. Inference latency < 100ms.
    Energy savings ≥ 20% per training round."

  The 4 contributions that make this thesis publishable in Q1/Q2:

  C1: First system jointly addressing hierarchy + DP + multimodal
      fusion + edge deployment with formal QoS guarantees.
      (No single prior paper addresses all four simultaneously.)

  C2: Privacy-preserving multimodal late fusion: modality-specific
      DP clipping per encoder branch — protects each sensor type
      independently while maintaining (ε=1, δ=1e-5)-DP.

  C3: 20× gradient compression (Top-k + 8-bit + error feedback) with
      <2% accuracy degradation under DP-SGD — communication saving
      beyond Phase 4's 76.4% to Phase 5 projected 95%.

  C4: End-to-end ONNX INT8 edge inference pipeline with measured
      P95 < 100ms — first FL paper to provide deployment-level
      inference latency validation (not just communication round time).
